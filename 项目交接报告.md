## 3. 项目目录怎么理解

你主要只用看这几个目录：

- `miniapp/`
  这是微信小程序前端
- `server/`
  这是 Node.js 后端
- `database/`
  这是 MySQL 脚本
- `knowledge/`
  这是问答知识库
- `docs/`
  这是项目说明文档

以前的 HTML 原型还在：

- `src/html_page/`

但它现在主要是参考，不是主开发目录。

## 4. 怎么启动

### 后端

在项目根目录打开终端后执行：

```powershell
cd server
npm install
npm run dev
```

如果只是想看配置和数据库是不是正常，可以执行：

```powershell
cd server
npm run doctor
```

后端启动后默认地址：

```text
http://localhost:3000
```

可以直接打开：

```text
http://localhost:3000/
http://localhost:3000/api/health
```

### 数据库

数据库脚本在：

- `database/schema.sql`
- `database/seed.sql`

如果本机还没初始化，就在 MySQL 里执行：

```sql
CREATE DATABASE IF NOT EXISTS carbon_agent DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE carbon_agent;
SOURCE C:/Users/何/Desktop/work/computer/大模型/项目/碳足迹Agent/database/schema.sql;
SOURCE C:/Users/何/Desktop/work/computer/大模型/项目/碳足迹Agent/database/seed.sql;
```

### 小程序

用微信开发者工具打开：

```text
miniapp/
```

注意两点：

- `project.config.json` 里的 `appid` 要和你当前使用的小程序一致
- 当前请求地址默认是 `http://localhost:3000/api`

所以：

- 开发者工具本地调试可以直接用
- 真机或上线必须换成正式 HTTPS 域名

## 6. 现在最缺什么

如果目标是“真的上线”，现在最缺的不是再画页面，而是这几件事：

1. 把后端部署到正式 HTTPS 域名
2. 把小程序请求地址改成线上域名并做真机联调
3. 补齐隐私政策、用户协议、审核材料
4. 做商城核销/发货后台
5. 做广场发帖、审核和运营后台
6. 优化 Agent 的知识库和检索效果

## 7. 建议你接手后的顺序

建议按这个顺序继续：

1. 先确认本机能完整跑起来
2. 再确认数据库真的在写入，不是 mock
3. 然后做部署和真机联调
4. 再补审核上线需要的合规内容
5. 最后再做后台和 Agent 增强

## 8. 交接时最值得先验证的几件事

你接手后，建议先自己走一遍下面这些：

1. `cd server && npm run dev`
2. 打开 `http://localhost:3000/api/health`
3. `cd server && npm run doctor`
4. 用微信开发者工具打开 `miniapp/`
5. 在小程序里试一次登录
6. 试一次日志打卡
7. 试一次商城兑换
8. 试一次广场认领和完成认领

这 8 步如果都通了，说明你已经接手成功。
